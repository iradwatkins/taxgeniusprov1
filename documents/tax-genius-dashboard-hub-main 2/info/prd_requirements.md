\# PRD Requirements

\#\#\# Functional

\* \*\*FR1:\*\* The platform must support a 3-tier preparer system (e.g., Trainee, Certified, Top Performer) with distinct roles, permissions, and dashboard views.  
\* \*\*FR2:\*\* The platform must provide features for a "Tax Genius Academy," including capabilities for hosting virtual training programs and tracking preparer certification status.  
\* \*\*FR3:\*\* The system must include CRM-like functionality to manage client leads, track follow-ups, and automate parts of the client onboarding journey.  
\* \*\*FR4:\*\* The system must allow certified preparers and referrers to create and manage custom vanity links (e.g., \`TaxGenius.com/YourName\`).  
\* \*\*FR5:\*\* The platform must feature a digital referral dashboard with analytics, leaderboards, and gamified bonuses to track and incentivize referrer performance.  
\* \*\*FR6:\*\* The system must support automated text and email drip campaigns for the entire client journey, from lead nurturing to upselling additional services.  
\* \*\*FR7:\*\* The platform must be architected to allow for the cross-selling of new services, such as credit repair, business incorporation, and bookkeeping.  
\* \*\*FR8:\*\* The universal login system must support multiple authentication methods: traditional email/password, passwordless magic links, and Google OAuth.  
\* \*\*FR9:\*\* The system must provide a secure, questionnaire-style workflow for clients to submit their tax information and upload necessary documents (e.g., IDs).  
\* \*\*FR10:\*\* The system must integrate with a third-party service (e.g., MarketingBoost.com) to automate the delivery of trip-based rewards for both clients and referrers.  
\* \*\*FR11:\*\* The platform must include a simple e-commerce storefront for selling branded merchandise to preparers and referrers.  
\* \*\*FR12:\*\* The system must allow users to generate custom, downloadable marketing posters that are embedded with their unique referral QR code.  
\* \*\*FR13:\*\* The platform must include an AI Agent integrated with models like Gemini or Claude to assist in generating SEO-compliant web page content for the 200+ localized landing pages. This agent will ensure each page includes an accordion section with relevant questions and answers. The AI-generated content must adhere strictly to the latest search engine standards.  
\* \*\*FR14:\*\* Following a successful tax return completion, the system shall automatically send an invitation to the client to join the referral program, clearly detailing the cash and trip-based rewards and the marketing tools provided.

\#\#\# Non-Functional

\* \*\*NFR1:\*\* The entire platform must be bilingual, supporting both English and Spanish, with automatic language detection based on the user's device settings.  
\* \*\*NFR2:\*\* The platform shall be developed as a Progressive Web App (PWA) using the Vite framework to ensure a responsive, mobile-first, and installable experience.  
\* \*\*NFR3:\*\* The system's architecture must be designed for high scalability to handle seasonal traffic peaks, particularly from January through April.  
\* \*\*NFR4:\*\* All public-facing content, especially the 200+ localized landing pages, must be architected and optimized for modern SEO and AI-driven search best practices.  
\* \*\*NFR5:\*\* The platform must be designed with the long-term architectural flexibility to eventually support a franchise model.
